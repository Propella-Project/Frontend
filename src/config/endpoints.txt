PROPELLA API ENDPOINTS
======================

Base URL: https://propella-api.vercel.app/api

AUTHENTICATION (JWT)
--------------------
POST /accounts/token/           - Login to get access/refresh tokens
  Payload: { "email": string, "password": string }
  Response: { "access": string, "refresh": string }

POST /accounts/token/refresh/   - Refresh access token
  Payload: { "refresh": string }
  Response: { "access": string }

POST /accounts/register/        - Register new user
  Payload: {
    "email": string,
    "password": string,
    "confirm_password": string,
    "username": string (optional),
    "nickname": string (optional),
    "referral_code": string (optional),
    "referred_by": string (optional)  // Referrer's code
  }
  Response: { "access": string, "refresh": string, "user": {...} }

POST /accounts/verify-email/    - Verify email with code
  Payload: { "email": string, "code": string }

POST /accounts/resend-code/     - Resend verification code
  Payload: { "email": string }

POST /accounts/forgot-password/ - Request password reset
  Payload: { "email": string }

PUT/PATCH /accounts/change-password/ - Change password (authenticated)
  Payload: { "old_password": string, "new_password": string }

POST /accounts/reset-password/{uid}/{token}/ - Reset password with uid/token
  Payload: { "new_password": string, "confirm_password": string }

Headers required for authenticated endpoints:
Authorization: Bearer <access_token>


USER MANAGEMENT
---------------
GET /accounts/all-users/        - Get all users (admin only)

PUT/PATCH /accounts/edit-user/{user_id}/ - Edit user by ID
  Payload: { "nickname": string, "email": string, ... }


EXAM PROFILE
------------
POST /accounts/create-exam-profile/ - Create exam profile
  Payload: {
    "nickname": string,
    "exam_date": string (YYYY-MM-DD),
    "study_hours_per_day": number,
    "ai_tutor_selected": string,
    "ai_voice_enabled": boolean
  }

PUT/PATCH /accounts/edit-exam-profile/{profile_id}/ - Edit exam profile
  Payload: Same as create


REFERRALS
---------
GET /accounts/referrals/        - Get referral stats for current user
  Response: {
    "user": {
      "id": string,
      "nickname": string,
      "referral_code": string,
      "referral_points": number,
      "total_referrals": number,
      "estimated_earnings": number
    },
    "referrals": [...]
  }

Note: 10 points = ₦30 (1 point = ₦3)


SUBSCRIPTIONS
-------------
GET /accounts/subscriptions/plans/ - Get available subscription plans

POST /accounts/subscriptions/subscribe/ - Subscribe to a plan
  Payload: { "plan_id": string, "payment_method": string }


API SCHEMA
----------
GET /api/schema/                - Get OpenAPI schema

ERROR HANDLING
==============

Standard HTTP status codes are used:
200 - Success
201 - Created
400 - Bad Request
401 - Unauthorized (token expired or invalid)
403 - Forbidden
404 - Not Found
500 - Server Error

Error response format:
{
  "message": string,
  "errors": { [field]: string[] } (optional, for validation errors)
}
